Workload Distribution Plan

Phase 1: Database Design (Everyone)

  • Have all members collaborate on the initial table specifications based on the provided ERD
  • This ensures everyone understands the full database structure before working on individual components

Phase 2: Development Tasks

Creates & Documentation

  • Develop the Creates.sql file for table creation
  • Take lead on documenting table structures
  • Help coordinate the overall database structure

Constraints & Drops

  • Develop the Constraints.sql file to implement all integrity constraints
  • Create the Drops.sql file for proper table removal
  • Focus on ensuring referential integrity is properly maintained

Insert Data & Testing

  • Create the Insert.sql file with test data (minimum 5 records per table)
  • Verify that inserts maintain referential integrity
  • Test that all constraints work properly with the test data

Queries & Integration

  • Develop the Queries.sql file with all required query types
  • Ensure queries work with the inserted test data
  • Test integration between all components

Phase 3: Review & Video Demonstration

  • Each member reviews everyone else's work to ensure consistency
  • All members collaborate on preparing the 10-minute demonstration video
  • Each member explains their portion of the work in the video
