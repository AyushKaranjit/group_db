Constraints Script (Approx. 2 minutes)

[3:00 - 3:45] Ayush: Thank you, Nabin. After creating the basic table structures, the next critical step is enforcing data integrity. This is done using the constraints_group1.sql script. (Show script briefly) Constraints are rules that ensure the accuracy and reliability of data in the database. We implemented several types.

  • Primary Keys (PK): Ensure each record in a table is unique (e.g., pk_students on student_id).
  • Foreign Keys (FK): Establish and enforce links between tables, ensuring referential integrity (e.g., fk_students_degrees linking students to degrees).
  • Check Constraints: Limit the value range that can be placed in a column (e.g., ensuring grade in student_submissions is one of the allowed letter grades, or status in attendances is 'P' or 'A').
  • Not Null Constraints: Ensure that a column cannot have a NULL value (e.g., student_fname cannot be empty).
  • Unique Constraints: Ensure that all values in a column or a set of columns are unique across the table (e.g., unique_assignment_code on code in assignments).
  • Default Constraints: Assign a default value to a column if no value is specified during insertion (e.g., setting default attendance_date to SYSDATE).

[4:45 - 5:00] Ayush: Applying these constraints ensures our database maintains logical consistency. Next, Utsab will discuss how we populated the tables with data.
