Video Demonstration Script Outline

Introduction (About 45 seconds)

Utsab: Good afternoon. We are Group 1: Nabin, Anuskar, Ayush, and myself, Utsab. We're presenting our database project, built from a provided ERD.

Utsab: Our core task involved creating SQL scripts to define the database structure, enforce data integrity, populate tables, and execute queries.

Utsab: We'll now walk through our development process. Nabin will start with table creation.

Creates Script (About 1 minute)

Nabin: Thank you, Utsab. I developed the scripts to create our database tables. The creates_group1.sql file (shows script) sets up the database structure based on the ERD, creating tables like students and modules. We defined clear column names and appropriate data types like NUMBER, VARCHAR2, and DATE`, and used SELECT * FROM tab; to verify table creation. This organization is key for a smooth-running database.

Nabin: Next, Ayush will discuss data constraints.

Constraints Script (About 2 minutes)

Ayush: Thanks, Nabin. Next, we added constraints using constraints_group1.sql. (shows script) These rules are vital for data quality and consistency, preventing bad data and protecting reliability. We used SELECT * FROM tab; to verify. Key constraints include:

  • Primary Keys: Ensures unique entries, like student_id.
  • Foreign Keys: Link tables and ensure valid relationships, like degree_id in students.
  • Check constraints: Ensure sensible data values, like a valid grade or module status.
  • Not Null constraints: Applied to essential fields like student_fname, ensuring important data isn't missed.
  • Unique constraints: Ensure unique values in non-primary key columns, like assignment_code.
  • Default values: Automatically used if no value is given, like attendance_date defaulting to SYSDATE.

Ayush: These constraints ensure an accurate and reliable database. Utsab will now explain data insertion.

Insert Script (About 1.5 minutes)

Utsab: Thank you, Ayush. With the structure and rules ready, we added sample data using inserts_group1.sql. (shows script) This script uses INSERT INTO commands for all tables, with ample realistic data to test functionality and constraints, verified with SELECT * FROM tab;.

Utsab: We used real-world examples for names and codes, handled dates as text like \'19-MAY-2025\' for Oracle's conversion, tested default values by omitting some columns, and carefully managed foreign key relationships, for instance, ensuring students and modules entries exist before adding to module_lists.

Utsab: Anuskar will now demonstrate data querying.

Queries Script (About 2 minutes)

Anuskar: Thank you, Utsab. To retrieve useful information, we used queries_group1.sql. (shows script) This demonstrates various SQL queries, including:

  • Simple Select Statements: Basic queries like SELECT * FROM students; to get all data from a table.
  • Projection Queries: Selecting specific columns, like student names and contact numbers.
  • Filtering with WHERE Clause: Using WHERE to find records meeting specific conditions, like students from a city.
  • Join Operations: Combining data from multiple tables, like students with their enrolled modules.
  • Advanced Queries with Aggregate Functions: Using functions like COUNT and AVG with GROUP BY for summary reports, like student counts per module.

Anuskar: These showcase the database's ability to turn data into insights. Nabin will now discuss the cleanup script.

Drops Script (About 30 seconds)

Nabin: I also created drops_group1.sql. (shows script) This script resets the database. Critically, it first removes all constraints, then deletes the tables, and finally purges the recyclebin. This order is vital for a clean slate, confirmed by SELECT * FROM tab;.

Nabin: Now, I'll explain the master run script.

Master Run Script (About 1 minute)

Nabin: Finally, to automate setup, we have run_group1.sql. (shows script) This master script sequentially runs all others: drops, then creates, constraints, inserts, and finally queries. If the database is already clear, the initial drops part might show \'table or view does not exist\' errors; this is normal and the script continues. This ensures consistent setup and testing with a single command.

Nabin: Utsab will now conclude.

Conclusion (About 30 seconds)

Utsab: In summary, our team successfully built the database system from the ERD, creating a full suite of SQL scripts for structure, rules, data, and queries.

Utsab: Our collaborative effort resulted in a complete, working system meeting all assignment requirements.

Utsab: Thank you.
